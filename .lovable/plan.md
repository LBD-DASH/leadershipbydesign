
# Fix Sitemap XML Format Error

## Problem
Google Search Console is rejecting your sitemap because when it requests `https://leadershipbydesign.co/sitemap.xml`, it receives an HTML page (your React app) instead of the actual XML file. This happens because the site is a Single Page Application (SPA), and the hosting platform routes all requests to the main app instead of serving static files directly.

## Solution
Point Google to the dynamic sitemap generated by your backend function, which correctly serves XML with the proper content-type headers.

## Changes Required

### 1. Update robots.txt
Change the sitemap URL from the static file to the backend function:

**Current:**
```text
Sitemap: https://leadershipbydesign.co/sitemap.xml
```

**Updated:**
```text
Sitemap: https://fdtlljyfjdlrymfeguoa.supabase.co/functions/v1/generate-sitemap
```

### 2. Add generate-sitemap to config.toml
Register the edge function so it can be called publicly without authentication.

### 3. Resubmit in Google Search Console
After publishing, resubmit the new sitemap URL in Google Search Console.

---

## Technical Details

The `generate-sitemap` edge function already exists and:
- Returns proper XML with `Content-Type: application/xml` header
- Includes all static pages
- Dynamically fetches published blog posts from the database
- Outputs valid XML sitemap format

No changes needed to the function itself - it's already correctly configured. We just need to:
1. Ensure it's registered in `config.toml` with `verify_jwt = false`
2. Point `robots.txt` to use it instead of the static file

---

## Expected Outcome
Google will successfully fetch and parse your sitemap, enabling proper indexing of all pages including dynamically added blog posts.
